# 盘前持仓检查开发落地说明 v0.1

> 目标：把持仓事实和人工维护的持仓计划字段用于每日盘前风险提示。

---

## 1. 定位

盘前持仓检查属于 `09 workflow` 的盘前准备，不是策略引擎，也不是自动交易。

第一版只做：

- 扫描当前持仓。
- 识别缺失字段和明显风险。
- 写入 SQLite。
- 在 Web 工作流页展示。

不做：

- 自动卖出。
- 自动下单。
- AI 重新判断。
- 修改持仓数量、成本或现金。

---

## 2. 第一版规则

规则：

- `MISSING_BUY_LOGIC`：缺原买入逻辑。
- `MISSING_EXIT_RULE`：缺证伪点或止损价。
- `T_PLUS_LOCKED`：存在 T+1 锁定数量。
- `NEAR_STOP_LOSS`：现价距离止损/证伪价不足 3%。
- `STOP_LOSS_BROKEN`：现价跌破止损/证伪价。
- `LOSS_WARNING`：浮亏超过 8%。
- `POSITION_TOO_HIGH`：当前仓位超过计划仓位或单票上限。
- `NEAR_TARGET_PRICE`：现价距离目标价不足 3%。
- `TARGET_REACHED`：现价达到目标价。
- `POSITION_CHECK_OK`：未发现明显风险。

---

## 3. 数据表

结果写入：

```text
position_pre_market_checks
```

字段包含：

- account_id
- symbol
- trade_date
- check_time
- category
- severity
- rule_code
- message
- current_price
- reference_price
- position_pct
- available_quantity
- locked_quantity

---

## 4. 使用方式

启动 Web 后进入：

```text
/workflows
```

点击：

```text
生成盘前持仓检查
```

结果会展示在“盘前持仓检查”区域。

处理状态：

- `UNREVIEWED`：未处理。
- `VIEWED`：已查看。
- `DEFER`：暂不处理。
- `NEED_DECISION`：需要生成进一步决策。
- `DONE`：已处理。
- `IGNORE`：忽略。

第一版只记录处理状态，不自动生成交易动作。

`生成决策` 会基于该检查项对应股票刷新数据，并生成持仓合并决策记录和报告。

注意：

- 生成决策不等于执行交易。
- 持仓股生成的是持仓/买入合并决策。
- 生成后检查项状态变为 `DECISION_CREATED`，页面可跳转查看决策详情。
